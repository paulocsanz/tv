# Oscar (Best Picture) Movies & Structured Awards Metadata

**Status:** Implemented

## Background

- The catalog had no structured awards data: `EnrichedItem.awards` is OMDb's free-text summary
  blurb (e.g. "Won 3 Oscars..."), rendered as-is on the title page. No per-nomination detail
  (category/year/won-vs-nominated), no way to browse "Oscar movies," and no data model that could
  support other events (festivals, etc.) later.
- The catalog is curated by hand (`data/top_400_curated.json` → `cargo run --bin enrich` →
  `backend/data/enriched_400.json`), so entire award lineages (e.g. every Best Picture nominee)
  were only present to the extent someone happened to add that specific title already.

## Problems This Solves

- **No way to browse or filter by award** — a user interested in Oscar history had no way to find
  those titles as a set.
- **Award info fragile at the title level** — the free-text OMDb blurb is unstructured, inconsistent
  in phrasing, and doesn't distinguish won vs. nominated, or say which category.
- **Missing catalog coverage** — most Best Picture nominees (as opposed to winners of Best Picture
  or Oscars in general, which skew toward already being "great movies" someone would've curated
  anyway) weren't in the catalog at all.

## Proposed Solution

- [x] **P0** Source a complete Best Picture nominee dataset (winners and losers) across every
      Academy Awards ceremony held to date, and add every title not already in the catalog.
      Implemented 2026-07-10: `data/oscars_best_picture.json`, 624 entries spanning all 98
      ceremonies (1929–2026, film years 1927–2025), compiled from Wikipedia's "Academy Award for
      Best Picture" article and cross-referenced against ceremony-date and per-film sources.
      Verified against 22 ceremonies' winners from general knowledge (all matched) plus structural
      checks (no duplicate title/year pairs, correct winner-per-ceremony counts, sane nominee
      counts per year). `resolve-oscars.js` matched nominees to the catalog locally by
      `(title, year)` first (avoiding trust in a catalog item's *stored* `tmdb_id`, which this repo
      has documented cases of being wrong for older/ambiguous titles), and only hit TMDB for the
      574 titles not already owned — 50 nominees were already in the catalog. `cargo run --bin
      enrich` (unmodified) enriched the new titles with full metadata.
- [x] **P0** Structured, per-nomination award metadata, generic across events. Implemented
      2026-07-10: `AwardEntry { event, category, year, won }` (`backend/src/models.rs`) and
      `EnrichedItem.award_entries: Vec<AwardEntry>`, populated via a side-file backfill
      (`backend/data/awards_backfill.json`, generated by `generate-awards-backfill.js`) merged onto
      items at boot by `apply_awards_backfill` (`backend/src/main.rs`) — the same pattern
      `keywords`/`collections` backfills already use. Only `event: "Academy Awards"` data is
      populated today; adding another event (Cannes, etc.) later is purely more rows in the same
      backfill file, no schema or code change. 624 nominations applied (99 won, 525 nominated —
      99 = 98 ceremonies + the 1st ceremony's parallel "Unique and Artistic Picture" award).
- [x] **P1** Make the new data browsable. Implemented 2026-07-10: `award_category`/`award_won`
      query params on `GET /api/content` (mirrors the existing `keyword` filter), and a
      "Best Picture Winners & Nominees" homepage row (`get_sections`).
- [x] **P1** Surface structured award badges on the title page alongside the existing free-text
      blurb. Implemented 2026-07-10: `frontend/app/title/[id]/page.tsx` renders a
      "🏆 Won"/"Nominated" pill per `award_entries` row.
- [x] **P2** Regression coverage for the awards backfill merge logic. Implemented 2026-07-10: unit
      tests in `backend/src/main.rs` (`backfill_tests` module) — merge-onto-matching-item,
      don't-overwrite-existing-entries, tolerate-missing-file, and a real-data sanity check that
      exercises the actual `enriched_400.json`/`awards_backfill.json` pair (no Postgres required,
      unlike the rest of this repo's test suite).

## Data-quality notes

- Sourcing caught and fixed its own bugs before they reached the catalog: `resolve-oscars.js`
  flags any TMDB match that required a title-only fallback or whose returned title differs
  substantially from the source title, into `backend/data/oscars_low_confidence_matches.json`. Of
  8 flagged entries, 7 were benign (subtitle/year variants of the correct film — e.g. TMDB listing
  "My Left Foot" as "My Left Foot: The Story of Christy Brown"). One was a real bug: TMDB's
  `/search/movie?query=Lion&year=2016` ranks "The Lion King" (1994) first by popularity despite the
  year filter — confirmed via direct API query, the real "Lion" (2016, Dev Patel/Garth Davis) is
  only the *second* result. Since `enrich.rs`'s own title+year search would have hit the identical
  ambiguity, "Lion" was added by direct TMDB-ID lookup instead (mirroring the manual
  TMDB-ID-verified fix pattern from commits `8dd489a`/`dafb436`), and the resolved mapping was
  corrected before generating the final backfill. Left in the catalog, the bug would have tagged
  the family classic "The Lion King" as a 2017 Best Picture nominee.
- Running the full enrich pipeline also resurfaced 3 titles previously removed for being
  duplicate/miscategorized entries (`fina-estampa-2020-tv`, `divine-love-2019-tv`,
  `once-upon-a-time-was-i-veronica-2012-movie` — see commit `e4a0632`): that earlier fix only
  edited `enriched_400.json`, not the curated source list, so any future enrich run would
  re-introduce them. Removed from both files this time so it doesn't recur.

## Known follow-ups (not addressed in this pass)

- This roughly doubled the movie count (255 → 829) and grew `/admin/catalog`'s "no torrent
  options" queue from 34 to 608 items. That page is an unpaginated flat list
  (`frontend/app/admin/catalog/page.tsx`) — fine at 34, not great at 608. Per the "metadata
  only, no acquisition" decision behind this feature, none of these were pushed through the
  torrent pipeline; they'll be picked up (or not) the same way any other catalog item is.
- Only Best Picture is populated. Every other Oscar category (acting, directing, foreign/
  international feature, documentary, etc.) and every other event (Cannes, Sundance, etc.) are
  explicitly out of scope for this pass, but the `AwardEntry` shape needs no changes to support
  them — just more sourcing + backfill-generation work later.
